# Session Reflections

Post-session analyses that drive toolkit evolution. Each file is a structured retrospective of one significant work session.

## Purpose

These reflections form a feedback loop: agent uses the toolkit → reflects on friction points → reflections inform toolkit improvements.

## How They're Created

After completing significant work in a session, the agent writes a reflection here. The trigger is either:
- The agent proposes it (per AGENTS.md instructions)
- The user requests it explicitly

## How to Use Them

Periodically review accumulated reflections to identify recurring patterns:

```
# Ask Claude to analyze multiple reflections at once:
"Read all files in reflections/ and summarize: what are the top 3 recurring
friction points? What toolkit changes would address them?"
```

## File Naming

`YYYY-MM-DD-brief-topic.md` — e.g., `2026-03-09-kinefx-rig-debug.md`
